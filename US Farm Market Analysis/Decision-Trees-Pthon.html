Decision Tree One

The ​first decision tree created by DecisionTreeClassifier function uses Entropy as the criterion, "best" as the splitter, the minimum samples split is set to be 2 and the minimum samples leaf is set to be 1.

SInce the tree is large, it's possible to predict this tree is overfitting.

Let's take a look at the heatmap built on the confusion matrix:

The heatmap from the confusion matrix okay. The size of the tree is 48, which is overfiiting since the amont of labels we have is 5.

Decision Tree Two

The second decision tree ​uses Gini measure, "best" as the splitter, the minimum samples split is set to be 10 and the minimum samples leaf is set to be 7. The tree size is 25 which is bigger than the label size 5, but it's better than the first one.

Let's take a look at the heatmap base on the confusion matrix:

This one is a little bit neater than the first one even though the tree size is still large. For now, this tree is better than the first one.

Decision Tree Three

The third decision tree ​uses Gini measure, "random" as the splitter, the minimum samples split is set to 15 and the minimum samples leaf is set to 45. The tree size is 4 which is smaller than the label size 5, so ti's possible that the prediction is not that accurrate.

Let's look at the heatmap based on the confusion matrix:

From the heatmap distribution, we can see that the third tree is not as nice as the first two.

The use of the heatmap leverages the confusion matrices. By analyzing the confusion matrices, it's easier to determining if the decision trees are effective in predicting data.
